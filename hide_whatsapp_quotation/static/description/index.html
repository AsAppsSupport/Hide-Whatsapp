Overview

What this module does

A lightweight UI patch that controls exactly when the WhatsApp button appears in the chatter, based on document state and type.

🚫

Hidden on Quotations

The WhatsApp button is automatically removed from the chatter while a Sales or Purchase document is still in Quotation / Draft stage.
✅

Visible on Confirmed Orders

As soon as a quotation is confirmed into a Sales or Purchase Order, the WhatsApp button reappears automatically — no manual steps needed.
🧩

Pure Frontend Patch

Implemented as a lightweight OWL component patch — no changes to backend logic, models, or the WhatsApp module itself.
⚡

Zero Configuration

Works immediately after install. No settings, no setup screens — the rule applies across all Sales and Purchase documents.

Behavior

How the button behaves

A simple state-based rule, applied consistently across Sales and Purchase documents.

1
Quotation / Draft

User opens a Sales or Purchase Quotation

While the document is still a draft Quotation, the chatter is rendered without the WhatsApp button — preventing accidental outreach before terms are finalized.
2
Confirmation

Quotation is confirmed into an Order

The moment the document state changes to a confirmed Sales Order or Purchase Order, the OWL patch re-evaluates and restores the button automatically.
3
Outcome

WhatsApp button is fully usable

From this point on, the WhatsApp button behaves exactly as standard Odoo intended — letting your team message customers once the order is real.
